## Technical Notes

1. **Message timing** - Use `videoTime` in seconds to trigger messages
2. **Random variance** - Add 1-3 second random delay to feel natural
3. **Batch loading** - Load messages in 5-minute batches for performance
4. **Pause handling** - If user pauses video, pause message scheduler
5. **Seek handling** - If user seeks forward, catch up messages; if backward, don't replay
